# 📊 Ad-Sentinel 통합 표준 미디어믹스 규격 (v1.1)

이 문서는 구글 시트(기획안)에서 수집할 데이터의 컬럼 순서와 형식을 정의합니다. 
모든 마케터는 매체 세팅 전 본 규격에 맞춰 기획안을 작성해야 합니다.

## 1. 데이터 계층 구조 (Hierarchy)
- **Level 1: 캠페인** (목적 및 총 예산)
- **Level 2: 세트/그룹** (일일 예산, 타겟팅, 기간)
- **Level 3: 광고(Ad)** (랜딩 URL, UTM, 소재명) -> **최종 검수 단위**

## 2. 컬럼 가이드라인 (Column Specifications)

| 컬럼명 | 형식 | 필수여부 | 설명 및 예시 |
| :--- | :--- | :---: | :--- |
| **매체 (Platform)** | Dropdown | 필수 | Meta / Google |
| **광고 계정 ID** | String | 필수 | Meta: `act_123...` / Google: `123-456-7890` |
| **캠페인명** | String | 필수 | 매체에 생성할 캠페인명과 100% 일치해야 함 |
| **세트/그룹명** | String | 필수 | Meta: 광고 세트명 / Google: 광고 그룹명 |
| **광고명 (Ad Name)** | String | 필수 | **소재 단위 이름 (랜딩/UTM 검수 기준점)** |
| **일일 예산** | Number | 필수 | 숫자만 입력 (예: 50000). 매체 통화 설정 확인 요망 |
| **시작일** | Date | 필수 | YYYY-MM-DD (예: 2026-03-01) |
| **종료일** | Date | 필수 | YYYY-MM-DD (예: 2026-03-31) |
| **최종 랜딩 URL** | URL | 필수 | UTM을 제외한 순수 연결 주소 (http/https 포함) |
| **UTM 파라미터** | String | 선택 | `utm_source=...` 형태로 시작하는 전체 스트링 |
| **픽셀/전환 ID** | String | 선택 | 검수할 Meta 픽셀 ID 또는 Google 전환 액션 ID |

## 3. 작성 시 유의사항 (Best Practices)

1. **이름 일치성:** 매체 API는 텍스트를 기준으로 데이터를 매칭합니다. 시트의 `캠페인/세트/광고명`에 오타나 앞뒤 공백이 있으면 '미세팅'으로 간주될 수 있습니다.
2. **URL 검증:** 랜딩 URL 컬럼에는 리다이렉트가 없는 최종 목적지 주소를 입력해 주세요. (시스템이 404 에러를 자동 체크합니다.)
3. **단위 환산:** 일일 예산 입력 시, 매체 계정 설정이 '달러($)'인지 '원(₩)'인지 반드시 확인 후 숫자를 기입하세요.
4. **광고 단위 확장:** 동일한 광고 세트 내에서 소재(이미지/영상)가 다를 경우, 각각 별도의 행(Row)으로 작성해야 개별 랜딩 URL 검수가 가능합니다.
