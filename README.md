# Ad-Sentinel (애드센티널)

**Ad-Sentinel**은 광고 기획안과 실제 매체에 세팅된 데이터를 실시간으로 비교 검수하여, 예산 누수나 세팅 오류를 방지하는 **광고 운영 자동 검수 대시보드**입니다. 

## 🎯 주요 목적 (Core Purpose)
디지털 광고 캠페인 운영 시, 담당자가 엑셀로 기획한 예산, 랜딩 URL, UTM 파라미터 등의 정보가 실제 매체(예: Meta Ads)에 정확히 세팅되었는지 **크로스 체크**하는 프로세스를 자동화합니다. 이를 통해 사람의 실수(Human Error)로 발생할 수 있는 막대한 금전적 손실이나 데이터 추적 오류를 사전에 차단합니다.

## ✨ 주요 기능 (Key Features)
1. **실시간 검수 센터 (Real-time Audit Center)**
   - 엑셀(xlsx) 기획안 파일 드래그 앤 드롭 업로드 지원
   - **확장 25개 컬럼 표준 템플릿 지원** 및 **ver1 (기본 검수) / ver2 (소재 검수) 모드 토글**
   - 헤드라인, 본문 카피, CTA(행동유도) 버튼 등 소재 카피 요소 검수 확장 (ver2 모드)
   - 타겟팅 요약, CBO(캠페인 예산) vs ABO(세트 예산), 과금 기준, 최적화 목표 등 세부 필드 검증
   - 매체 API(Meta Graph / Google Ads API)를 호출하여 기획안 데이터와 Live 데이터의 글자 단위 불일치 검수 (랜딩 URL, UTM 등)
   - **지면 매핑 가이드(Placement Mapping Guide) 탭 및 인라인 팝업**을 통한 지면 키워드(`[페이스북 (피드)]` 등) 및 다중 URL 작성 가이드 제공
   - 결과에 따른 직관적인 상태 뱃지 (PASS / WARNING / FAIL) 제공

2. **검수 히스토리 (Audit History)**
   - 과거 검수 내역 및 발생했던 오류, 에러 로그 영구 기록
   - 남은 오류 내역을 일괄 해소할 수 있는 "오류 내역 All Pass" 기능 통해 히스토리 트래킹 편의성 강화

3. **팀/사용자 및 권한 설정 (Settings & Admin)**
   - Supabase Auth 및 RLS(Row Level Security)를 이용한 완벽한 데이터 격리 및 권한 관리(SUPER_ADMIN, ADMIN, USER)
   - 팀원 추가, 삭제 및 소속 관리
   - 매체 연동 관리: Meta 등 각 매체별 API 플랫폼 토큰을 DB에 안전하게 암호화 보관 및 로컬 `.env` 환경 Fallback 지원

## 🛠 기술 스택 (Tech Stack)
- **Framework:** Next.js (App Router), React
- **Styling:** Tailwind CSS, Lucide React (Icons)
- **Database & Auth:** Supabase (PostgreSQL, Row Level Security)
- **Utilities:** `xlsx` (Excel Parsing), `date-fns` (Date Formatting)

## 🗄 데이터베이스 스키마 (Core Data Structure)
Ad-Sentinel은 Supabase (PostgreSQL) 기반으로 구축되어 있으며, 주요 테이블은 다음과 같습니다. 해당 구조는 코어 비즈니스 로직과 강하게 결합되어 있으므로 임의 수정 시 주의가 필요합니다.

1. **`users`**: 사용자 계정 정보 (`auth.users`와 1:1 매핑) 및 팀 소속(`team_id`), 권한(`role: SUPER_ADMIN, ADMIN, MEMBER`) 관리
2. **`teams`**: 대행사 또는 부서 단위의 팀 그룹
3. **`team_account_map`**: 팀과 실제 매체(Meta/Google) 광고 계정(`ad_account_id`)을 매핑하는 N:M 연결 테이블
4. **`platform_settings`**: 팀별 또는 전역 매체 API 연동 정보 보관 (App ID, Secret, Refresh Token 등)
5. **`planned_campaigns` / `live_campaign_settings`**: 기획안(Excel) 데이터와 실제 매체(Live) 데이터 캐싱 및 이력 보관용
6. **`audit_logs`**: 실시간 검수 결과 오류/경고 발생 내역 및 해결(Resolved) 상태 추적
7. **`ad_enum_values`**: 각 매체(Meta, Google)의 캠페인 목적, 입찰 전략 등 열거형(Enum) 값과 한글 명칭, 설명이 매핑된 참조(Reference) 테이블

## 🏗 핵심 아키텍처 규칙 (Architecture Guidelines)
- **RLS (Row Level Security)**: Supabase RLS가 전면 적용되어 있습니다. 클라이언트 및 API Route(서버 액션)에서 일반 `supabase` 클라이언트를 사용할 경우 로그인한 유저의 소속 팀(`team_id`) 데이터만 조회/수정 가능합니다.
- **Service Role Bypass**: `teams` 생성이나 `team_account_map` 할당과 같이 RLS `INSERT/DELETE` 권한이 제한된 작업은 서버 액션(`use server`) 내에서 권한(`SUPER_ADMIN/ADMIN`) 검증 후, 별도의 `adminClient`(Service Role Key 사용)를 통해 처리합니다.
- **매체 연동 로직 (`actions.ts`)**: Meta Graph API 및 Google Ads API 통신은 보안상 철저하게 서버 환경(Server Actions)에서만 수행되며, Client-side에서는 호출 결과만 받아 UI를 렌더링합니다.
- **캐싱 및 상태**: React `useTransition`을 활용하여 UI Optimistic Updates와 서버 동기화를 매끄럽게 처리합니다.

## 📊 실시간 대시보드 (Real-time Dashboard) 지표 가이드

대시보드의 라이브 캠페인 목록은 현재 매체에 라이브(ENABLED/ACTIVE) 중인 캠페인의 **예산 소진율**과 **진행 상태**를 실시간으로 모니터링합니다.

### 1. 전체 예산과 일 예산의 적용 기준

매체(Meta, Google)의 API 응답에 따라 캠페인(혹은 광고 세트)의 총 예산을 식별하는 방식입니다. 대시보드 상의 '예산 소진율(%)'은 이 **총 예산(Total Budget)**을 분모로 하여 계산됩니다.

| 예산 세팅 유형 | 대시보드 적용 로직 (분모 산정 방식) |
|---|---|
| **전체 예산 (Lifetime Budget)** | 캠페인 세팅 시 '전체 예산'이 설정된 경우, 해당 값을 총 예산으로 즉시 사용합니다. |
| **일 예산 (Daily Budget) + 종료일 있음** | 전체 예산이 설정되지 않고 일 예산만 있는 경우, 캠페인의 **총 운영 일수(종료일 - 시작일 + 1)**를 구하여 일 예산에 곱한 **'가상 총예산'**을 산출하여 분모로 사용합니다. |
| **상시 운영 (종료일 없음)** | 일 예산으로 진행되나 '종료일'이 지정되지 않은 상시 운영 캠페인의 경우, 총 예산을 계산할 수 없으므로 대시보드 상에서 소진율은 **'예산 미설정'**으로 표기됩니다. |

### 2. 예산 소진 상태 (Burn Status) 판정표

'예산 소진 상태'는 캠페인이 운영된 시간적 비율(**기간 진행률**)과 실제 비용이 사용된 비율(**예산 소진율**)을 대조하여, 페이스에 맞춰 예산이 잘 소진되고 있는지를 시각적 뱃지로 알려주는 지표입니다.

> **계산 공식**: 오차율 = (예산 소진율 %) - (기간 진행률 %)

| 상태 뱃지 | 오차율 범위 | 의미 및 판정 기준 |
|:---:|:---:|---|
| 🟢 **● 정상** | `-15% ~ +15%` | 캠페인 기간 진행률과 예산 소진율이 오차 범위(±15%) 내에서 안정적으로 비례하여 소진 중인 상태 |
| 🟡 **▼ 미소진** | `-15% 미만` | 시간은 지났으나 예산이 그에 못 미치게 소진된 상태 (예: 기간은 50% 지났으나 예산은 20%만 소진됨) |
| 🔴 **▲ 과소진** | `+15% 초과` | 시간이 지난 것에 비해 예산이 과도하게 빨리 소진된 상태 (예: 기간은 20% 지났으나 예산은 50%가 소진됨) |
| ⚪ **— 기간 미설정** | `-` | 캠페인 종료일이 설정되지 않아 기간 진행률 및 오차율을 계산할 수 없는 캠페인 |

### Meta Ads 검수 항목

> **매칭 기준**: 광고 세트명 기준으로 매칭 (캠페인 ID 입력 시 ID 우선 → 없으면 캠페인명으로 매칭)

| # | 항목 | 검수 방식 | 판정 |
|---|---|---|:---:|
| 1 | **광고 세트 존재 여부** | 템플릿 `광고 세트명` ↔ 매체 광고 세트 목록 비교 | FAIL |
| 2 | **통화** | 템플릿 `통화` ↔ 계정 통화 코드 | FAIL |

| 4 | **광고 세트 예산 (ABO)** | 캠페인 예산 미입력 시 `광고 세트 일/전체 예산` ↔ 매체 세트 예산 비교 | FAIL |
| 5 | **시작일** | 템플릿 `시작일` ↔ 매체 `start_time` (날짜 앞 10자리 비교) | FAIL |
| 6 | **종료일** | 템플릿 `종료일` ↔ 매체 `stop_time` | FAIL |
| 7 | **캠페인 목적** | 템플릿 `캠페인 목적` ↔ 매체 `objective` | FAIL |
| 8 | **구매 유형** | 템플릿 `구매 유형` ↔ 매체 `buying_type` | FAIL |
| 9 | **최적화 목표** | 템플릿 `최적화 목표` ↔ 매체 `optimization_goal` | FAIL |
| 10 | **과금 기준** | 템플릿 `과금 기준` ↔ 매체 `billing_event` | FAIL |

| 13 | **광고 존재 여부** | 템플릿 `광고명` ↔ 해당 세트 내 매체 광고명 | FAIL |
| 14 | **랜딩 URL** | 템플릿 `랜딩 URL` ↔ 매체 광고 link (도메인+경로 비교, 쿼리스트링 제외) | FAIL |
| 14‑1 | **랜딩 URL 미설정** | 매체 광고에 URL이 아예 없는 경우 | WARNING |
| 15 | **UTM 파라미터** | 템플릿 `UTM 파라미터` ↔ 매체 `url_tags` (포함 여부 비교) | FAIL |

| 6 | **캠페인 목적(채널타입)** | 템플릿 `캠페인 목적` ↔ 매체 `advertising_channel_type` (예: SEARCH, SHOPPING, PERFORMANCE_MAX) | FAIL |
| 7 | **광고 그룹 존재 여부** | 템플릿 `광고 세트명` ↔ 해당 캠페인 내 광고그룹명. **PMax는 미입력 시 스킵** | FAIL |

| 9 | **랜딩 URL** | 템플릿 `랜딩 URL` ↔ 매체 `final_urls[0]` (도메인+경로 비교) | FAIL |
| 9‑1 | **랜딩 URL 미설정** | 매체 광고에 final_url이 없는 경우 | WARNING |
| 10 | **UTM 파라미터** | 템플릿 `UTM 파라미터` ↔ 매체 `tracking_url_template` (포함 여부 비교) | FAIL |

## 📋 엑셀 템플릿 입력 편의 기능

### Fill-down (값 자동 승계)

동일 캠페인 하위에 여러 광고 세트 / 광고가 있는 경우, **첫 번째 행에만 공통값을 입력**하고 나머지 행은 해당 셀을 비워두면 위 행의 값이 자동으로 승계됩니다.

#### 승계 대상 필드

| 레벨 | 자동 승계 필드 |
|---|---|
| **캠페인** | 매체, 팀명, 계정 ID, 캠페인 ID, 캠페인명, 통화, 캠페인 일예산, 캠페인 예산, 시작일, 종료일, 캠페인 목적, 구매 유형 |
| **광고 세트** | 광고 세트명, 광고 세트 일예산, 광고 세트 예산, 최적화 목표, 과금 기준, 픽셀/이벤트, 이벤트 유형 |
| **광고** (승계 제외) | 광고명, 랜딩 URL, UTM 파라미터 — 광고 단위 고유값이므로 반드시 직접 입력 |

#### 입력 예시

```
# 수정 전: 모든 행에 캠페인 공통값을 중복 입력해야 했음
매체  | 캠페인명  | 광고 세트명 | 광고명
Meta  | 캠페인 A  | 세트 1      | 소재_1
Meta  | 캠페인 A  | 세트 1      | 소재_2   ← 캠페인명/세트명 반복 입력
Meta  | 캠페인 A  | 세트 2      | 소재_3   ← 캠페인명 반복 입력

# 수정 후: 첫 행만 입력, 빈 칸은 위 행 값 자동 승계
매체  | 캠페인명  | 광고 세트명 | 광고명
Meta  | 캠페인 A  | 세트 1      | 소재_1
      |           |             | 소재_2   ← 캠페인명·세트명 자동 승계
      |           | 세트 2      | 소재_3   ← 캠페인명만 승계, 세트명은 새로 입력
```

> **주의**: 셀에 `0`을 명시적으로 입력한 경우에는 승계되지 않습니다. 오직 셀을 완전히 비워둔 경우에만 위 행 값이 승계됩니다.

## 📑 25개 컬럼 템플릿 명세 및 검수 모드 (대안 B)

Ad-Sentinel은 기본 필수 컬럼 외에도 소재 카피 및 지면별 다중 URL/UTM 검수까지 지원하는 **25개 컬럼 확장 템플릿(대안 B)**을 제공합니다.

### 1. 검수 모드 토글 (ver1 / ver2)

실시간 검수 센터 상단에서 **ver1**과 **ver2** 검수 모드를 자유롭게 전환할 수 있습니다.

- **ver1 (중요 세팅 내역서)**: 캠페인/세트 예산, 시작일/종료일, 캠페인 목적, 구매 유형 등 **상위 예산 및 일정/목적 핵심 세팅 내역** 중심의 검수 모드입니다. (소재 카피 요소 제외)
- **ver2 (세부 세팅 내역서)**: 예산 및 일자 항목 대신 **헤드라인, 본문 카피, CTA(행동유도) 버튼** 등 광고 소재 단위의 세부 미디어 카피라이팅 및 랜딩 URL, UTM, 최적화/과금/픽셀 항목에 집중하는 세부 검수 모드입니다.

---

### 2. 직관적 데이터 비교 레이아웃 (계획 VS API 실제)

- **깔끔한 검수 결과 (PASS / FAIL / WARN)**: "검수 결과" 셀에 복잡한 장문의 에러 텍스트를 줄이지 않고, 깔끔한 뱃지만 표시하여 시각적 가독성을 높였습니다.
- **셀 단위 필드별 대조 카드**: 각 항목 셀(예산, 시작일/종료일, 목적, 구매유형, 헤드라인, 본문카피, CTA, 랜딩URL, UTM 등)에 **`계획:` (엑셀 기획안 값)** 과 **`실제:` (Live API 연동 값)** 을 나란히 표시합니다.
- **불일치 시각 강조**: 세팅값 차이(오차) 발생 시 해당 셀이 **분홍/빨간색 테두리 카드(`❌ [불일치 사유]`)**로 강조 표시되어 불일치 항목을 한눈에 식별할 수 있습니다.

### 3. 지면 매핑 가이드 (Placement Mapping Guide) & Multi-Placement 작성 규칙

하나의 광고 세트/광고 내에서 매체 지면(Placement)별로 서로 다른 랜딩 URL이나 UTM 파라미터를 사용하는 경우, **지면 키워드**를 활용하여 다중 URL을 기입할 수 있습니다.

#### 💡 지면 가이드 탭 & 인라인 팝업
- **용어 사전 메뉴 > 지면 매핑 가이드 탭**: Meta 및 Google Ads 매체별 엑셀 입력 키워드, API 매핑값, 권장 소재 사이즈 및 가이드를 한눈에 조회할 수 있습니다.
- **검수 센터 인라인 팝업**: 검수 화면 상단의 `[지면 가이드]` 버튼을 누르면 검수 중 즉시 팝업으로 가이드를 확인하고 원하는 키워드의 `[예시 복사]` 버튼을 눌러 클릭 한 번으로 복사할 수 있습니다.

#### 📝 Multi-Placement 엑셀 작성 예시
엑셀 셀 내에서 `Alt + Enter` (줄바꿈)을 사용하여 지면 그룹별 URL/UTM을 구분 기입합니다. 대괄호 `[...]` 안의 지면 키워드는 가이드에 지정된 명칭을 정확히 작성해야 파서가 해당 지면을 올바르게 대조합니다.

```text
# 랜딩 URL 셀 작성 예시 (Alt+Enter로 줄바꿈)
[기본 설정 / 인스타그램 (스트림) / 페이스북 (피드)] https://www.example.com/promo?utm_source=facebook&utm_medium=feed
[페이스북 (릴스) / 페이스북 (스토리)] https://www.example.com/promo?utm_source=facebook&utm_medium=story
```

#### 📌 대표 지면 입력 키워드 예시

| 매체 | 엑셀 입력 키워드 (`[...]` 내 기입) | API 실제 값 | 권장 소재 사이즈 |
|:---:|---|---|---|
| **Meta** | `페이스북 (피드)` | `facebook_feed` | 1:1 (1080x1080) 또는 4:5 |
| **Meta** | `인스타그램 (피드)` | `instagram_feed` | 1:1 (1080x1080) 또는 4:5 |
| **Meta** | `인스타그램 (스토리)` | `instagram_story` | 9:16 (1080x1920) |
| **Meta** | `페이스북 (릴스)` | `facebook_reels` | 9:16 (1080x1920) |
| **Google**| `구글 (검색)` | `SEARCH` | 텍스트 소재 |
| **Google**| `구글 (디스플레이)` | `DISPLAY` | 가로형 (1200x628) / 정사각형 |
